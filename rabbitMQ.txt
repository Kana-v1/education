Exchanges:
Direct exchange - кидает сообщение в очередь которая определяется на основе message routing key
Fanout exchange - кидает сообщения во все очереди, с которыми он связан игнорит message routing key
Topic exchange - кидает сообщения в 1 или несколько топиков  на основе соответствия message routing key и паттерна по которому топики к нему привязываются (короче шо ключ в сообщении == ключу в очереди).
Header exchange - пишет в очереди на основе инфы в хэдерах.  

Queue:
Name - ...
Durable - сколько места на диске занимает
Exlusive - удаление очереди когда она не нужна
Auto delete - удаляет очередь когда консьюмер отписывается

Bindings - связывает exchange с очередью. У него (байндинга) может быть routing key attribute который используется некоторыми exchanges чтобы роутить сообщения. Их смысл в том чтобы кидать ОПРЕДЕЛЕННЫЕ сообщения в нужную очередь.

Если сообщение не может быть никуда записано то оно или дропнется или вернется в паблишеру в зависимости от настроек
