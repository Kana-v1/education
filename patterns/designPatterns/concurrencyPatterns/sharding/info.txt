Раскидывает данные из большой структуры по небольшим партициям чтобы задоджить локи при чтении/записи

Vertical sharding - данные из большой струкруты распихиваются по 2+ структурам, которые по сути являются 
частью большой структуры данных

Horizontal sharding - разнос данных по разным инстансам сервиса

Partitions: 
 - Shard - lockable collection represeting a single data partition
 - ShardedMap - обертка над шардом которая позволяет читать/писать так, как будто Sharded - 
 простая мапа
